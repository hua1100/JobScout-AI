## 使用範例

### 只搜尋台北市
```bash
AREA_CODES="6001001000"
```

### 搜尋雙北地區(台北+新北)
```bash
AREA_CODES="6001001000,6001002000"
```

### 搜尋北北基(台北+新北+基隆)
```bash
AREA_CODES="6001001000,6001002000,6001003000"
```

### 搜尋北部(台北、新北、桃園、新竹)
```bash
AREA_CODES="6001001000,6001002000,6001004000,6001005000,6001006000"
```

### 搜尋六都(台北、新北、桃園、台中、台南、高雄)
```bash
AREA_CODES="6001001000,6001002000,6001004000,6001008000,6001013000,6001016000"
```

### 搜尋全台灣(不限地區)
```bash
# 方法1: 留空
AREA_CODES=""

# 方法2: 註解掉
# AREA_CODES="6001001000,6001002000"
```

## 設定步驟

1. **開啟 `.env` 檔案**
2. **找到 `AREA_CODES` 設定項**
3. **填入你想要的地區代碼** (用逗號分隔)
4. **儲存檔案**
5. **執行爬蟲**

## 進階技巧

### 組合關鍵字 + 地區

假設你想找:
- **關鍵字**: AI自動化、RPA
- **地區**: 只要雙北

在 `.env` 中設定:
```bash
SEARCH_KEYWORDS="AI自動化,RPA"
AREA_CODES="6001001000,6001002000"
SCRAPY_PAGES_PER_KEYWORD=10
```

這樣會搜尋:
- AI自動化 + 台北市 (10頁)
- AI自動化 + 新北市 (10頁)
- RPA + 台北市 (10頁)
- RPA + 新北市 (10頁)

總共約 800 筆職缺 (每頁約20筆)

## 驗證設定

執行爬蟲時,會在LOG中看到:

```
搜尋關鍵字: AI自動化, AI轉型, 數位轉型, 流程自動化, RPA, AI工程師
地區篩選: 6001001000, 6001002000
每個關鍵字爬取: 5 頁
```

如果看到「地區篩選: 不限地區(全台灣)」,表示沒有設定地區代碼。

## 提示

💡 **建議做法:**
1. 先用小範圍測試(1-2個關鍵字 + 1個地區 + 1-2頁)
2. 確認結果符合預期後再擴大範圍
3. 避免一次爬取太多資料(容易被封鎖)

⚠️ **注意事項:**
- 地區代碼必須完全正確,否則會被忽略
- 多個地區用「逗號」分隔,不要有空格
- 如果爬不到資料,可能是該地區+關鍵字組合沒有職缺
